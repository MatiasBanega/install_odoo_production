El módulo Odoo Restart permite al administrador INICIAR, DETENER y REINICIAR el servidor de odoo mediante los comandos para iniciar, detener y reiniciar el servidor. El administrador puede administrar varios servidores y puede agregar detalles como el nombre de usuario, el nombre del cliente, la URL o IP del servidor, el puerto y la contraseña. Esto resulta útil cuando el cliente enfrenta dificultades durante la necesidad de actualizar varias acciones del servidor. Estas son las características de este módulo:

  • El administrador puede listar las credenciales para el servidor de diferentes clientes.
  • Además, como los comandos de acción (start, stop, restart) del servidor varían según el sistema operativo, el administrador puede configurar los comandos para INICIAR / DETENER / REINICIAR según corresponda.
  • Cuando se hace clic en el botón de acción del servidor, el comando mencionado se ejecutará con éxito implementando la acción.
  • El administrador también puede rastrear las acciones del servidor realizadas para varios servidores cliente a través de la función de gestión del historial.
  • Se puede mencionar información adicional en la pestaña de instrucciones si es necesario.

Requisitos previos antes de la instalación

  • pexpect: sudo pip3 install pexpect.

Configuraciones

  • Solo los usuarios con derecho de acceso a "Odoo Restart" pueden administrar las acciones del servidor.
  • Ajustes > Usuario > Other > Odoo Restart
  • Agregar detalles del servidor.

Como puede ver en la imagen, el administrador puede agregar varios servidores cliente con varios detalles, como nombre de usuario, contraseña, URL del servidor, puerto y nombre del cliente. Se crean grupos de usuarios separados para que la acción del servidor se pueda administrar fácilmente y el administrador también puede proporcionar acceso a un usuario en particular para administrar la acción del servidor.

Vaya a Odoo Restart-> Crear-> Agregar detalles.

Luego de listar los detalles del servidor cliente, el administrador, en caso de cualquier dificultad del servidor cliente, puede realizar varias acciones del servidor, como INICIAR, DETENER y REINICIAR. Además, el administrador puede proporcionar comandos para las acciones del servidor según el sistema operativo.

Cuando se hace clic en la acción del servidor, el comando particular se ejecutará en el backend y el problema del servidor del cliente se resolverá fácilmente sin necesidad de conectarse al terminal Odoo.

Todas las acciones realizadas se pueden gestionar a través del historial. El administrador puede ver el nombre del servidor cliente, creado por qué usuario y la acción del servidor realizada en ese servidor.
